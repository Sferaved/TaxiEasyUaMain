2. Старый способ vs Google Pay

2.1. Для пользователя

Критерий Карта WFP (wfp_payment) Google Pay (google_pay_payment)
Подготовка Привязка карты в WebView WayForPay (rectoken) Не нужна — карта из Google Wallet
Момент оплаты Заказ сразу; холд после создания заказа на сервере Сначала шторка GPay и холд; заказ только после успеха
Подтверждение При заказе часто без отдельного шага (карта уже привязана) Каждый заказ: отпечаток / PIN / «Оплатить»
Сумма холда Полная стоимость заказа Полная стоимость заказа

2.2. Технически (для тестировщика)

ШагКарта по токенуGoogle Pay
1orderReference → wfpInvoiceТо же
2orderClientCostMyApi → заказ в БДПока не вызывается
3 orderIdMemoryToken + chargeActiveToken(rectoken) googlePayConfig → шторка → googlePayCharge(gpToken)
4WFP: холд по rectokenWFP: холд по токену GPay
5Poll / push статусаПосле успеха → тот же orderClientCostMyApi
6—orderIdMemoryToken, poll — без chargeActiveToken
Важно: при GPay второй холд по активной WFP-карте (rectoken) не выполняется — иначе было бы двойное списание.

2.3. Поток Google Pay (кратко)

«Заказать» → googlePayConfig → Шторка GPay → googlePayCharge → Успех WFP → orderClientCostMyApi

2.4. Вилка «безнал + нал» в диспетчере

Поведение как у карты по токену:

  • authorizationChoiceApp('google_pay_payment') → payment_type = 1 (безнал).
  • При настроенной вилке: безнальная нога + наличная нога (алгоритм поиска).
  • Это не два списания: один холд GPay + две заявки в диспетчере.

Исключение: только my_server_api (оператор недоступен) — один заказ без вилки, как у карты.

2.5. TEST vs PRODUCTION

УсловиеРежим GPayЧто видит пользователь
Эмулятор TEST Test Card •••• 1111, текст про тестовое средовище
Телефон + debug APK TEST То же
Телефон + release APK PRODUCTION Реальная карта Wallet, без предупреждения о тесте

В logcat: WfpGooglePayHelper: Google Pay environment: TEST или PRODUCTION.

Приёмка на реальном устройстве — только release 1.1030+, не debug из Android Studio.

3. Требования перед тестом

3.1. Устройство

  • Google Play Services и Google Wallet.
  • Банковская карта Visa/Mastercard в Wallet.
  • Стабильный интернет.

3.2. Приложение

  • PAS_1 1.1030+, сборка release.

3.3. Сервер

  • Деплой taxi_repo с google_pay_payment (коммит 5606fb3d).
  • WayForPay: Google Pay включён для merchant города.

3.4. В приложении

  • Город с безналом (как для WFP-карты).
  • Способ оплаты: Google Pay.

4. Проверка на реальном устройстве

Сценарий A — успешный заказ

  1. Visicom → маршрут → стоимость.
  2. Способ оплаты: Google Pay.
  3. «Заказать» → шторка GPay: ваша карта, сумма верна, нет Test Card / тестового предупреждения.
  4. Подтвердить оплату.
  5. Ожидание: поиск машины, заказ безнал в диспетчере; при вилке — обе ноги.

Сценарий B — отмена GPay

  1. Шаги 1–4 сценария A, затем закрыть шторку.
  2. Заказ не создан, остаётесь на экране заказа.

Сценарий C — нет двойного холда

  1. WFP-карта привязана, оплата — Google Pay.
  2. Один холд GPay, нет charge по rectoken.

Сценарий D — не тестируем в 1.1030

Не проверятьПочему
HomeFragmentGPay при заказе не включён
Доплата через GPayЭтап 3
Смена на GPay в шторке без заказаТолько запись в БД

6. Эмулятор

  • GPay всегда TEST — это нормально.
  • Холд WFP с TEST-токеном часто падает (1129) — не провал release-приёмки.
  • На эмуляторе: шторка, отмена, отсутствие заказа при ошибке.
